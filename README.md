1. 정렬 알고리즘 (Sorting Algorithms)
데이터를 특정 순서로 배열하는 알고리즘. 버블 정렬 O(n²), 퀵 정렬 O(n log n), 병합 정렬 O(n log n) 등이 있다. 시간 복잡도와 안정성(stable) 여부가 선택 기준.

2. 스택과 큐 (Stack & Queue)
스택은 LIFO(후입선출), 큐는 FIFO(선입선출) 구조. 스택은 함수 호출, 괄호 검사에 사용. 큐는 BFS, 작업 스케줄링에 활용.

3. 이진 탐색 (Binary Search)
정렬된 배열에서 중간값과 비교하며 탐색 범위를 절반씩 줄이는 알고리즘. 시간 복잡도 O(log n). 정렬이 전제 조건.

4. 해시 테이블 (Hash Table)
키를 해시 함수로 변환해 값을 저장하는 자료구조. 평균 O(1) 탐색/삽입. 충돌 해결 방법으로 체이닝, 오픈 어드레싱 사용.

5. 그래프 탐색 (BFS/DFS)
BFS는 너비 우선으로 큐 사용, 최단 거리 탐색에 적합. DFS는 깊이 우선으로 스택/재귀 사용, 경로 탐색이나 백트래킹에 활용.

6. 동적 프로그래밍 (Dynamic Programming)
큰 문제를 작은 부분 문제로 나누고, 중복 계산을 메모이제이션으로 방지. 최적 부분 구조와 중복 부분 문제 조건 필요.

7. 트리 (Tree)
계층적 자료구조. 이진 트리, 이진 탐색 트리(BST), AVL, 레드블랙 트리 등. 순회 방식: 전위/중위/후위/레벨 순회.

8. 시간 복잡도 (Time Complexity)
알고리즘 효율성 측정. Big-O 표기법 사용. O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) 순으로 느려짐.
